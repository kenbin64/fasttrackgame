‚ö° 5-Minute Quickstart

The Core Idea

In dimensional computing, objects already contain all their parts, properties, and behaviors. You don't construct them ‚Äî you invoke them through dimensional levels.

from helix import HelixKernel, Substrate

# Create the kernel and substrate
kernel = HelixKernel()
substrate = Substrate()
kernel.attach(substrate)

# Create a token (entity) at spiral 0
car = substrate.add_token("car", spiral=0)

# Navigate through dimensional levels
kernel.invoke(0)  # Level 0: Potential (void)
kernel.invoke(1)  # Level 1: Point (identity exists)
kernel.invoke(2)  # Level 2: Length (properties emerge)
kernel.invoke(6)  # Level 6: Whole (fully materialized)

# Materialize tokens at current level
tokens = kernel.materialize()
print(tokens)  # ‚Üí {Token(car, level=6)}
import { HelixKernel, Substrate } from '@butterflyfx/helix';

// Create the kernel and substrate
const kernel = new HelixKernel();
const substrate = new Substrate();
kernel.attach(substrate);

// Create a token (entity) at spiral 0

// Navigate through dimensional levels

kernel.invoke(1);  // Level 1: Point (identity exists)
kernel.invoke(2);  // Level 2: Length (properties emerge)
kernel.invoke(6);  // Level 6: Whole (fully materialized)

// Materialize tokens at current level
const tokens = kernel.materialize();
console.log(tokens);  // ‚Üí Set { Token(car, level=6) }

üåÄ Helix State Machine

The Helix Kernel is a formal state machine with state space:

H = {(s, ‚Ñì) | s ‚àà ‚Ñ§, ‚Ñì ‚àà {0,1,2,3,4,5,6}}

Where:

  • s ‚Äî Spiral number (which turn of the helix, can be negative)
  • ‚Ñì ‚Äî Level (dimensional layer 0-6)

Think of it as a DNA helix: you can move up through levels within a spiral, or spiral up to the next turn, or spiral down to the previous turn.

‚óã

Immutable State

HelixState is frozen ‚Äî operations return new states, never mutate.

‚àû

Infinite Spirals

Spirals extend infinitely in both directions. Level 6 ‚Üí Spiral Up ‚Üí Level 0.

‚éã

O(7) Complexity

Maximum 7 operations per spiral. No iteration, no scanning.

üìä The 7 Dimensional Levels

0: Potential 1: Point 2: Length 3: Width 4: Plane 5: Volume 6: Whole

‚öôÔ∏è Kernel Operators

Operator Signature Description
INVOKE(k) (s, ‚Ñì) ‚Üí (s, k) Jump to level k within current spiral
SPIRAL_UP (s, 6) ‚Üí (s+1, 0) Move to next spiral (requires level 6)
SPIRAL_DOWN (s, 0) ‚Üí (s-1, 6) Move to previous spiral (requires level 0)
COLLAPSE (s, ‚Ñì) ‚Üí (s, 0) Return to potential (idempotent)

Python API

from helix import HelixKernel

kernel = HelixKernel()

# INVOKE - jump to a level
kernel.invoke(3)         # (0, 0) ‚Üí (0, 3)
print(kernel.state)      # ‚Üí (0, 3:Width)

# SPIRAL_UP - go to next spiral
kernel.invoke(6)         # Must be at level 6
kernel.spiral_up()       # (0, 6) ‚Üí (1, 0)
print(kernel.state)      # ‚Üí (1, 0:Potential)

# SPIRAL_DOWN - go to previous spiral
kernel.spiral_down()     # (1, 0) ‚Üí (0, 6)
print(kernel.state)      # ‚Üí (0, 6:Whole)

# COLLAPSE - return to potential
kernel.collapse()        # (0, 6) ‚Üí (0, 0)
print(kernel.state)      # ‚Üí (0, 0:Potential)
‚ö†Ô∏è Invariants:
‚Ä¢ Level must always be 0-6
‚Ä¢ SPIRAL_UP requires level=6
‚Ä¢ SPIRAL_DOWN requires level=0
‚Ä¢ COLLAPSE is idempotent (can call repeatedly)

üß¨ Substrate & Tokens

The Substrate is the manifold where tokens live. Tokens are entities with:

  • location ‚Äî Coordinates in the manifold (spiral, level, position)
  • signature ‚Äî Unique identity across all dimensions
  • payload ‚Äî Lazy data, only dereferenced at materialization
from helix import Substrate, Token, PayloadSource

substrate = Substrate()

# Create a token with stored payload
car = substrate.add_token(
    name="car",
    spiral=0,
    payload={"make": "Tesla", "model": "Model S"}
)

# Create a token with GEOMETRIC payload (derived from manifold)
wave = substrate.add_token(
    name="wave",
    spiral=0,
    source=PayloadSource.GEOMETRIC,
    property=GeometricProperty.SIN  # Value = sin(position)
)

# Materialization (Œº function)
kernel.attach(substrate)
kernel.invoke(6)
tokens = kernel.materialize()
# Returns all tokens at current spiral and level

üíª Code Examples

Example 1: Car Simulation

Model a car where properties emerge as you invoke levels:

# Level 0: Car is potential (not yet observed)
kernel.invoke(0)
# Nothing materialized

# Level 1: Identity exists
kernel.invoke(1)
# car.id = "vehicle_001"

# Level 2: Properties emerge
kernel.invoke(2)
# car.speed, car.fuel, car.temperature

# Level 3: Relations to other entities
kernel.invoke(3)
# car.lane, car.nearby_vehicles

# Level 4: Full context plane
kernel.invoke(4)
# car.road, car.traffic_state

# Level 5: Internal depth
kernel.invoke(5)
# car.engine.pistons, car.internals

# Level 6: Complete whole
kernel.invoke(6)
# car is fully manifested, ready for interaction

Example 2: Dimensional Query

Find entities without iteration:

# Traditional: O(n) scan
for piston in engine.pistons:
    if piston.misfiring:
        return piston

# Dimensional: O(1) direct access
engine.pistons.where(misfiring=True)

Example 3: Multi-Path Access

Same object, multiple valid paths:

# All of these reference the SAME spark plug:
car.engine.cylinder[2].sparkPlug
car.sparkPlugs.where(cylinder=2)
car.sparkPlugs[2]

# Drill up or down freely:
atom.electrode.sparkPlug.piston.engine.car  # up
car.engine.piston[0].sparkPlug.atoms[0]     # down
atom.car                                     # skip levels
